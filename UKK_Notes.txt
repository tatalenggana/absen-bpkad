Penjelasan Website absen-bpkad

Website absen-bpkad ini adalah sebuah sistem absensi online yang dibangun menggunakan framework Laravel. Fungsinya adalah untuk mencatat kehadiran (check-in/check-out) pegawai atau anggota BPKAD, serta memungkinkan admin untuk memantau laporan kehadiran. Fitur utamanya kemungkinan termasuk:
- Autentikasi (login/register) untuk user dan admin.
- Check-in dan check-out dengan pencatatan waktu dan lokasi.
- Laporan kehadiran untuk admin.
- Pengaturan (settings) terkait absensi.

---

Penjelasan Komponen Laravel (Middleware, Models, Controllers)

Laravel menggunakan arsitektur MVC (Model-View-Controller) yang memisahkan logika aplikasi menjadi tiga bagian utama.

1. Middleware:
    - Apa itu: Middleware adalah lapisan kode yang berjalan sebelum atau sesudah sebuah request HTTP sampai ke controller. Mirip seperti "penjaga gerbang" atau "filter" untuk setiap permintaan yang masuk ke aplikasi Anda.
    - Fungsi:
        - Otentikasi: Memastikan user sudah login sebelum bisa mengakses halaman tertentu. (Contoh: AuthMiddleware)
        - Otorisasi: Memastikan user memiliki peran atau izin yang benar (misal: hanya admin yang bisa akses halaman admin). (Contoh: AdminMiddleware, UserMiddleware)
        - Logging: Mencatat setiap request yang masuk.
        - Manipulasi Request/Response: Mengubah data request sebelum ke controller atau response sebelum dikirim balik.
    - Di Projectmu: Kamu punya AdminMiddleware.php dan UserMiddleware.php di folder app/Http/Middleware, yang berarti website ini menggunakannya untuk membedakan akses antara admin dan user biasa.

2. Models:
    - Apa itu: Model adalah representasi dari data di database Anda. Di Laravel, kita sering menggunakan Eloquent ORM (Object-Relational Mapping) untuk berinteraksi dengan database. Model bertindak sebagai jembatan antara aplikasi dan tabel di database.
    - Fungsi:
        - Interaksi Database: Menyimpan, membaca, mengubah, dan menghapus data dari tabel database.
        - Logika Bisnis: Bisa mengandung aturan atau logika terkait data itu sendiri.
        - Relasi Data: Mendefinisikan hubungan antar tabel (misal: satu user punya banyak data kehadiran).
    - Di Projectmu: Kamu punya Attendance.php, User.php, UserProfile.php di folder app/Models. Ini artinya kamu punya tabel attendances, users, dan user_profiles di database yang dikelola oleh masing-masing Model ini.

3. Controllers:
    - Apa itu: Controller adalah bagian yang menerima dan menangani permintaan (request) dari user. Ini adalah otak yang mengkoordinasikan apa yang harus dilakukan setelah menerima request.
    - Fungsi:
        - Menerima Request: Menganalisis permintaan user (misal: "mau lihat laporan kehadiran").
        - Mengambil Data: Memanggil Model untuk mendapatkan atau menyimpan data dari database.
        - Memproses Data: Melakukan logika bisnis sederhana (misal: menghitung total kehadiran).
        - Mengembalikan Response: Menentukan apa yang akan dikirim kembali ke browser user (misal: menampilkan view HTML, mengirim data JSON).
    - Di Projectmu: Kamu punya AttendanceController.php, AuthController.php, dan Controller.php di folder app/Http/Controllers. Ini berarti ada controller khusus untuk mengelola absensi, autentikasi user, dan controller dasar.
